# Auth & Access Control System (MVP)

## Описание

Это упрощённая (урезанная) версия backend-приложения с собственной системой аутентификации и авторизации.

Основная цель — показать подход к построению системы доступа, а не реализовать её в полном объёме.

---

# Архитектура

Система разделена на два уровня:

## 1. Аутентификация

Определяет, кто пользователь.

Реализовано:

* регистрация
* логин
* logout
* деактивация пользователя (soft delete)
* сессии и токены

---

## 2. Авторизация

Определяет, что пользователь может делать.

Используется модель RBAC (роли + права).

---

# Структура доступа

## Основные сущности

### User

* пользователь системы
* имеет одну роль

---

### Roles

* ADMIN
* MANAGER
* USER
* GUEST

---

### BusinessElement

Абстрактные ресурсы системы (например: USERS, PRODUCTS).

---

### AccessRolesRules

Связывает:

* роль
* ресурс
* права доступа

---

# Модель прав

Права реализованы через битовую маску:

```id="c2z9gf"
READ_OWN     = 0b0000001
READ_ALL     = 0b0000010
CREATE       = 0b0000100
UPDATE_OWN   = 0b0001000
UPDATE_ALL   = 0b0010000
DELETE_OWN   = 0b0100000
DELETE_ALL   = 0b1000000
```

Проверка:

```id="6j8d2k"
bool(byte_flag & PERMISSION_BIT)
```

---

# Логика работы

1. Определяется пользователь (по токену или сессии)
2. Получается его роль
3. Находится правило доступа
4. Проверяется нужное право

---

# Важно: ограничения реализации

Данная версия намеренно упрощена и не доведена до полной реализации.

В частности:

* используется одна роль на пользователя
* нет централизованного сервиса проверки прав
* частично используется функциональность DRF
* не реализована полностью логика owner-based доступа
* отсутствует кеширование

---

# Причина упрощения

Проект выполнен как MVP:

* чтобы показать структуру системы
* чтобы продемонстрировать понимание RBAC
* чтобы не усложнять реализацию раньше необходимости

---

# Возможное развитие

При необходимости система может быть расширена:

* добавить PermissionService
* сделать несколько ролей на пользователя
* вынести всё в кастомный middleware
* добавить кеширование
* реализовать полную проверку прав (own/all)

---

# Вывод

Проект представляет собой базовую реализацию системы аутентификации и авторизации с заделом на расширение.

Основная задача — продемонстрировать архитектурный подход, а не законченный продукт.
